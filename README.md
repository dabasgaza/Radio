# Radio

نظام مكتبي لإدارة العمل الإذاعي اليومي داخل بيئة إنتاج محلية، مبني بـ `WPF` و `.NET 10` مع فصل واضح بين الواجهة، منطق الأعمال، والوصول إلى البيانات.

المشروع يركّز على إدارة:

- البرامج
- الحلقات
- الضيوف
- المراسلين
- التغطيات
- المستخدمين والصلاحيات
- سجلات التنفيذ والنشر
- التقارير التشغيلية

## نظرة عامة

يوفّر المشروع واجهة تشغيل عربية لسيناريو عمل إذاعي يعتمد على دورة حياة واضحة للحلقة:

`مجدولة -> منفذة -> منشورة`

كما يتضمن صلاحيات على مستوى الوظائف، وحذفاً منطقياً للسجلات، وتسجيل تغييرات على مستوى قاعدة البيانات عبر `EF Core SaveChangesInterceptor`.

## لقطات معمارية

الحل مقسّم إلى ثلاث طبقات رئيسية:

- `Radio`
  واجهة المستخدم المكتبية بـ `WPF`، النوافذ، الشاشات، والموارد البصرية.
- `DataAccess`
  الخدمات، DTOs، التحقق من المدخلات، منطق الصلاحيات، وتنسيق الوصول إلى البيانات.
- `Domain`
  الكيانات، الـ `DbContext`، وضبط النماذج والعلاقات مع `EF Core`.

هذا الفصل يجعل المشروع أسهل في الصيانة والتوسعة، ويقلل من تداخل مسؤوليات الطبقات.

## الميزات الحالية

- واجهة عربية مخصّصة لسطح المكتب باستخدام `WPF`
- تسجيل دخول وربط المستخدم بالدور الوظيفي
- إدارة البرامج والحلقات والضيوف والمراسلين والتغطيات
- إدارة المستخدمين ومصفوفة الصلاحيات
- تتبع حالة الحلقة من الجدولة حتى النشر
- حذف منطقي للسجلات عبر `IsActive`
- تسجيل تغييرات قاعدة البيانات عبر `AuditInterceptor`
- استخدام `IDbContextFactory` لإنشاء `DbContext` قصير العمر
- رسائل UI موحدة للتنبيهات والتأكيدات
- شاشات تقارير وإحصاءات تشغيلية
- المشروع حاليا لا يستخدم MVVM

## الوحدات الوظيفية

### 1. إدارة البرامج

إنشاء البرامج، تعديلها، وعرضها داخل الواجهة مع ربطها بالحلقات.

### 2. إدارة الحلقات

إضافة الحلقات، جدولة وقت التنفيذ، ربطها بالبرنامج والضيف، وتحديث حالتها التشغيلية.

### 3. إدارة الضيوف

حفظ بيانات الضيوف وتحديثها وربطهم بالحلقات.

### 4. إدارة المراسلين والتغطيات

متابعة بيانات المراسلين والتغطيات المرتبطة بهم حسب الموضوع والموقع.

### 5. إدارة المستخدمين والصلاحيات

إضافة المستخدمين، تعديل أدوارهم، وتخصيص الصلاحيات حسب الوظيفة.

### 6. التنفيذ والنشر

تسجيل خطوات التنفيذ والنشر وربطها بالحلقات داخل النظام.

## آلية العمل

يعتمد المشروع على مجموعة مبادئ عملية داخل الكود:

- الخدمات في `DataAccess/Services` تحتوي منطق الأعمال الأساسي
- واجهة المستخدم تستدعي الخدمات مباشرة عبر `Dependency Injection`
- التحقق من المدخلات يتم مركزياً في `ValidationPipeline`
- الصلاحيات يتم فحصها باستخدام `UserSession` و `AppPermissions`
- الحذف المنطقي يمر عبر `BaseEntity.IsActive`
- تغييرات البيانات يتم التقاطها في `AuditInterceptor`

## التشغيل المحلي

1. استنسخ المستودع:

```bash
git clone https://github.com/dabasgaza/Radio.git
```

2. افتح الملف:

```text
Radio.slnx
```

3. عدّل سلسلة الاتصال داخل:

```text
Radio/appsettings.json
```

مثال:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=BroadcastWorkflowDB;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}
```

4. جهّز قاعدة البيانات بما يتوافق مع الجداول المطلوبة في المشروع.

5. نفّذ البناء:

```bash
dotnet build Radio.slnx
```

6. شغّل مشروع الواجهة `Radio`.
