## 📝 نبذة عن المشروع
**بث برو** هو نظام مؤسسي (Enterprise-Grade) صُمم خصيصاً لإدارة العمليات اليومية في الإذاعات المحلية والقنوات الرقمية. يغطي النظام دورة حياة المحتوى الإذاعي بالكامل؛ بدءاً من جدولة البرامج والحلقات، مروراً بإدارة الضيوف والمراسلين الميدانيين، وصولاً إلى توثيق التنفيذ والإنتاج والنشر الرقمي.

---

## ✨ المميزات الجوهرية (Key Features)

### 🛡️ أمن وصلاحيات متقدمة
*   **نظام صلاحيات ديناميكي:** مصفوفة صلاحيات (Permissions Matrix) تسمح لمدير النظام بالتحكم في كل فعل (إضافة، تعديل، حذف، تنفيذ، نشر) لكل دور وظيفي على حدة.
*   **تشفير BCrypt:** حماية قصوى لكلمات مرور المستخدمين باستخدام أقوى خوارزميات التشفير الحديثة.

### 📝 تتبع التدقيق الآلي (Automated Audit Trail)
*   **معترض ذكي (Interceptor):** يقوم النظام تلقائياً بتسجيل أي تغيير يحدث في قاعدة البيانات، مع تخزين القيم القديمة والجديدة بصيغة **JSON**، مما يتيح معرفة "من قام بتغيير ماذا ومتى" بدقة متناهية.

### 💬 نظام رسائل مركزي (Ambient Context Messaging)
*   **إشعارات سلسة:** استخدام الـ **Snackbar** للرسائل العابرة والـ **DialogHost** لرسائل التأكيد (نعم/لا) بشكل Thread-safe، مما يضمن تجربة مستخدم دون نوافذ منبثقة مزعجة.

### 🔄 إدارة دورة حياة الحلقة
*   **Workflow صارم:** تتبع حالة الحلقة من (مجدولة ← منفذة ← منشورة) مع فرض قواعد عمل تمنع تجاوز المراحل أو التلاعب بالبيانات بعد النشر.

### 🗑️ الحذف المنطقي العالمي (Global Soft Delete)
*   **حماية البيانات:** لا يتم حذف أي سجل نهائياً من القاعدة؛ بل يتم تعليم السجلات كـ "غير نشطة"، مع فلترة تلقائية تمنع ظهورها في أي مكان في النظام بفضل الـ **Global Query Filters**.

## 🏗️ المعمارية التقنية (Technical Architecture)

تم بناء النظام باتباع معايير هندسية صارمة لضمان الأداء والقابلية للتوسع:

*   **النمط البرمجي:** Code-Behind احترافي مع فصل كامل للمنطق في طبقة الخدمات (Services Layer).
*   **إدارة البيانات:** استخدام `IDbContextFactory` لضمان سياقات قاعدة بيانات قصيرة العمر (Short-lived Contexts) تمنع تضارب البيانات في بيئات العمل الجماعية.
*   **معالجة التزامن:** حماية البيانات من التعديل المتزامن (Optimistic Concurrency) عبر حقل `RowVersion` وعرض نوافذ مقارنة ذكية للمستخدم في حال حدوث تعارض.
*   **التخزين المؤقت (Caching):** استخدام `IMemoryCache` لتسريع جلب البيانات شبه الثابتة (الأدوار، الصلاحيات، الحالات).

## 🚀 كيفية التشغيل (Setup)

1.  **قاعدة البيانات:** قم بتنفيذ سكريبت SQL الموجود في مسار `/Database/Final_Schema.sql`.
2.  **الإعدادات:** عدل نص الاتصال (Connection String) في ملف `appsettings.json`.
3.  **المتطلبات:** تأكد من تثبيت .NET 9 SDK و Visual Studio 2022 (الإصدار الأخير).
4.  **الدخول الافتراضي:**
    *   **اسم المستخدم:** `superadmin`
    *   **كلمة المرور:** `Admin@2025`
